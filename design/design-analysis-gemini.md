# Overview & Architecture Analysis

The project is a modern full-stack application built with **Next.js 15**, following a **GraphQL-centric architecture**. It integrates frontend and backend layers into a unified codebase using the Next.js App Router.

## Core Architectural Patterns

1. **GraphQL API Layer**
    - **Unified Endpoint**: All data operations flow through a single GraphQL endpoint at `/api/graphql`.
    - **Schema-First Development**: The backend defines its structure using `typeDefs.ts` (schema) and `resolvers.ts` (logic), ensuring a clear contract between the frontend and backend.
    - **Context Injection**: The GraphQL server (in `route.ts`) extracts user information from JWT tokens to provide a secure execution context for every request.

2. **Type-Safe Persistence (Drizzle ORM)**
    - **Schema as Code**: The database schema is defined in TypeScript (`src/db/schema.ts`), providing end-to-end type safety from the database to the UI.
    - **SQLite/LibSQL**: Uses a lightweight, performant SQL engine ideal for both development and edge deployment.

3. **Client-Side Data Management (URQL)**
    - **Reactive UI**: Uses **URQL** as the GraphQL client to handle data fetching, caching, and state synchronization.
    - **Auth Integration**: A custom `GQLProvider` automatically injects the stored JWT token into outgoing request headers, handling secure communication transparently.

4. **State & Authentication**
    - **JWT-Based Auth**: Authentication is handled via JSON Web Tokens, with logic centralized in `src/utils/auth.ts` and `bcrypt` used for password security.
    - **React Context**: User session state is managed globally using `UserContext.tsx`, allowing components to easily access the current user's status.

## Directory Structure Highlights

- `src/app/api/graphql/`: The heart of the backend, containing resolvers and schema definitions.
- `src/db/`: Database configuration and Drizzle schema definitions.
- `src/gql/`: Centralized storage for GraphQL queries and mutations used by the frontend.
- `src/app/components/`: Modular React components (Sidebar, TopBar, Issue, etc.) following a component-driven UI pattern.
- `migrations/`: SQL migration files generated by Drizzle to track database changes.

## Summary of Data Flow
A user action (e.g., creating an issue) triggers a **URQL mutation**. This mutation is sent as a POST request with a **JWT token** to the **Next.js API route**. The **GraphQL resolver** validates the request, uses **Drizzle ORM** to update the **SQLite database**, and returns the result back to the frontend to update the UI state.
